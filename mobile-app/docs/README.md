# GrandHR mobile — compliance docs

Store-submission and compliance documents kept next to the code so
they version together with the app.

| File | Purpose |
|---|---|
| [`PRIVACY_POLICY.md`](./PRIVACY_POLICY.md) | Source of truth for the policy published at `https://grandhr.in/privacy-policy`. |
| [`TERMS_OF_SERVICE.md`](./TERMS_OF_SERVICE.md) | Source of truth for `https://grandhr.in/terms`. |
| [`DATA_SAFETY.md`](./DATA_SAFETY.md) | Copy/paste answers for the Google Play **Data safety** form. |
| [`ACCOUNT_DELETION.md`](./ACCOUNT_DELETION.md) | Copy to put on `https://grandhr.in/account-deletion` (Play Store requirement). |
| [`STORE_LISTING.md`](./STORE_LISTING.md) | Short + long descriptions, screenshots checklist, content-rating answers. |
| [`RELEASE_CHECKLIST.md`](./RELEASE_CHECKLIST.md) | Run-book for every production release. |
| [`IOS_REVIEW_NOTES.md`](./IOS_REVIEW_NOTES.md) | Goes verbatim in App Store Connect → App Review notes. |

When you change what the app collects or how it behaves, update every
file that touches the same subject. The `RELEASE_CHECKLIST.md` has a
step for this.
